# Man in the middle attack vulnerability 
The first problem I had with my app, was noticing the data wasn't really encrypted. That is a serious issue I had to solve before moving on to development. As you can see, using Wireshark, I can with  ease eavesdrop on conversation between client and server. Let's say client writes to server "secret info" and server writes it back:

As you can see, it's unacceptable.
# Solution
I researched, and found a solution to this critical vulnerablity. I needed TLS encryption, so that even if an attacker capture the content, it would be unreadable. I generated myself a self signed key(even tough it's self-signed, it's still secure as it uses modern and strong cryptographic alghoritms. The only difference between signed byself and signed by someone trusted, is that browsers wouldn't let people connect through them. But for local development, it's okay. I'll change it later. 
```bash
openssl req -x509 -newkey rsa:2048 \
-keyout server.key \
-out server.crt \
-days 365 \
-nodes \
-subj "/CN=localhost"
```
This generated me a `server.key` file and `server.crt` file. 
`server.key` is a private key, that the server will use to decrypt data sent by user. It's private, so I can't share it here, but you can generate your own and use it. 
`server.crt` is a certificate self-signed by me. It is for letting client check if the server is genuine. I shared mine for example here, but you can generate your own and use it. (Remember it's only for local testing. Self signed certificate won't work in whole web, as browsers won't trust it).
